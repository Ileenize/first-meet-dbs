# Meeting First Game Option

## 1. Describe the human problems
[ปัญหาของผู้เล่น]
- ผู้เล่นบอร์ดเกมจำนวนมากไม่ได้มีปัญหาแค่ว่า “ไม่มีโต๊ะ” หรือ “ไม่รู้จะเล่นเกมอะไร” แต่ปัญหาหลักคือ ไม่มีคนเล่นด้วย 
- โดยเฉพาะเกมที่ต้องใช้จำนวนผู้เล่นหลายคน เช่น 4–6 คนขึ้นไป หากจำนวนคนไม่ครบ เกมก็ไม่สามารถเริ่มได้ หรือเล่นได้ไม่สนุกตามที่ควรจะเป็น
- อีกปัญหาคือการนัดหมายผ่านแชททั่วไป เช่น LINE Group Facebook Group มักไม่มีระบบยืนยันที่ชัดเจน ทำให้เกิดปัญหา “นัดแล้วเท” หรือมีคนยกเลิกกะทันหัน ส่งผลให้ตี้ล่ม ผู้เล่นที่ตั้งใจไปเสียเวลา และคนที่เหลือไม่สามารถเล่นเกมตามแผนได้
- นอกจากนี้ ผู้เล่นใหม่ยังอาจรู้สึกกังวลเมื่อต้องเดินเข้าไปในคาเฟ่บอร์ดเกมโดยไม่รู้จักใคร ไม่รู้ว่ามีตี้ว่างหรือไม่ ไม่รู้ว่าเกมไหนเหมาะกับตัวเอง และไม่มั่นใจว่าจะสามารถเข้าร่วมกับคนแปลกหน้าได้หรือไม่ ปัญหานี้ทำให้ผู้เล่นบางคนไม่กล้าเริ่มเล่นบอร์ดเกม แม้จะมีความสนใจก็ตาม

[ปัญหาของร้านคาเฟ่บอร์ดเกม]
- ฝั่งร้านคาเฟ่บอร์ดเกมมีปัญหาเรื่องการจัดการโต๊ะและจำนวนลูกค้า ร้านอาจมีลูกค้าจองโต๊ะไว้ แต่สุดท้ายไม่มาตามนัด ทำให้ร้านเสียโอกาสในการให้บริการลูกค้าคนอื่น โดยเฉพาะช่วงเวลาที่มีลูกค้าเยอะ เช่น วันศุกร์ เสาร์ อาทิตย์ หรือช่วงเย็น
- ร้านยังขาดระบบกลางที่ช่วยให้เห็นข้อมูลการจองล่วงหน้าอย่างเป็นระบบ เช่น วันนี้มีตี้กี่กลุ่ม แต่ละกลุ่มมีกี่คน ใช้โต๊ะไหน เล่นช่วงเวลาใด และมีผู้เล่นยืนยันแล้วกี่คน 
- หากไม่มีระบบจัดการที่ดี ร้านต้องพึ่งการจดมือ แชท หรือโทรศัพท์ ซึ่งทำให้ข้อมูลกระจัดกระจายและผิดพลาดได้ง่าย


## 2. Describe how your solution solves these problems
- แอปพลิเคชันนี้จะแก้ปัญหาโดยทำหน้าที่เป็น Platform Matchmaker & Booking System สำหรับผู้เล่นบอร์ดเกมและร้านคาเฟ่บอร์ดเกม 
- โดยไม่เน้นเพียงการจองโต๊ะ แต่เน้นการสร้างและเข้าร่วม ตี้เล่นบอร์ดเกม
- ผู้เล่นสามารถค้นหาตี้ที่เปิดรับคนอยู่ในแต่ละคาเฟ่ ดูจำนวนคนที่ต้องการ เวลาที่เล่น ระดับผู้เล่น และประเภทของตี้ได้ เช่น ตี้ที่ระบุเกมชัดเจน หรือ ตี้แบบเปิดที่ยังไม่ระบุเกมและให้พนักงานร้านช่วยแนะนำเกมให้
- ระบบจะช่วยลดปัญหานัดล่มด้วยการให้ผู้เล่นจ่ายเงินมัดจำก่อนเข้าร่วมตี้ เมื่อจ่ายมัดจำแล้ว สถานะการจองจะเปลี่ยนเป็น confirmed ทำให้ผู้จัดตี้และร้านมั่นใจได้ว่าผู้เล่นมีความตั้งใจมาจริง 
- หากผู้เล่นมาตามนัด ร้านสามารถสแกน QR Code เพื่อยืนยันการเข้าร่วม และเงินมัดจำอาจถูกเปลี่ยนเป็นส่วนลดเครื่องดื่มหรือค่าบริการภายในร้าน 
- แต่หากผู้เล่นไม่มาตามนัด ระบบสามารถบันทึกสถานะเป็น flaked และเงินมัดจำอาจถูกริบตามเงื่อนไขของระบบ
- สำหรับร้านคาเฟ่ ระบบจะมี dashboard ให้เจ้าของร้านหรือพนักงานดูรายการตี้ที่จองเข้ามา ดูจำนวนผู้เล่นที่ยืนยันแล้ว ดูโต๊ะที่ถูกใช้งาน และอัปเดตสถานะโต๊ะได้ 
- ทำให้ร้านจัดการพื้นที่และลูกค้าได้แม่นยำขึ้น ลดปัญหาโต๊ะว่างจากการจองแล้วไม่มา และเพิ่มโอกาสในการขายอาหาร เครื่องดื่ม หรือค่าบริการเล่นเกม

## 3. Describe your E-commerce business idea including core features
- แนวคิดธุรกิจของโปรเจคนี้คือ: Marketplace for Board Game Meetups
- แพลตฟอร์มตัวกลางที่เชื่อมระหว่าง ผู้เล่นที่ต้องการหาเพื่อนเล่นบอร์ดเกม กับ คาเฟ่บอร์ดเกมที่ต้องการบริหารโต๊ะและเพิ่มจำนวนลูกค้า
- ธุรกิจนี้ไม่ได้ขายสินค้าโดยตรง แต่ขาย “ประสบการณ์การเล่นบอร์ดเกมร่วมกับผู้อื่นในสถานที่จริง” 
- โดยมีระบบจอง ระบบมัดจำ และระบบยืนยันการเข้าร่วมเป็นกลไกหลัก
- รายได้ของแพลตฟอร์มสามารถมาจากค่าธรรมเนียมธุรกรรม เช่น หักเปอร์เซ็นต์เล็กน้อยจากเงินมัดจำหรือค่าจอง และอาจมีรายได้จากเงินมัดจำที่ถูกริบกรณีผู้เล่นไม่มาตามนัด โดยแบ่งรายได้กับร้านค้าตามโมเดลธุรกิจที่กำหนดไว้

## 4. Scope of Project
- ผู้เล่นสามารถหาตี้ได้
- สร้างตี้ได้
- เข้าร่วมตี้ได้
- จ่ายมัดจำแบบจำลองได้
- ร้านสามารถดูและจัดการ booking ได้

โปรเจคนี้จะพัฒนาเว็บแอปพลิเคชันสำหรับการค้นหา สร้าง และเข้าร่วมตี้เล่นบอร์ดเกมที่คาเฟ่ โดยมีผู้ใช้งาน 2 กลุ่มหลัก ได้แก่ ผู้เล่นทั่วไปและเจ้าของร้านคาเฟ่บอร์ดเกม

ระบบจะให้ผู้เล่นสามารถสมัครสมาชิก เข้าสู่ระบบ ค้นหาตี้ สร้างตี้ เข้าร่วมตี้ และจ่ายเงินมัดจำแบบจำลองเพื่อยืนยันการเข้าร่วม หลังจากจ่ายเงินสำเร็จ ระบบจะสร้าง booking code หรือ QR Code สำหรับใช้ check-in ที่ร้าน

ฝั่งเจ้าของร้านสามารถจัดการข้อมูลร้าน โต๊ะภายในร้าน รายการบอร์ดเกม และดู dashboard ของตี้ที่ถูกจองในร้านได้ รวมถึงสามารถยืนยันการมาถึงของผู้เล่นผ่าน booking code หรือ QR Code

ระบบจะใช้ MongoDB แบบ reference-heavy โดยแยก collections หลัก เช่น Users, Cafes, CafeTables, BoardGames, CafeGames, Meetups, Bookings และ Payments เพื่อให้จัดการข้อมูลได้ง่าย และสอดคล้องกับโครงสร้างของระบบ booking และ payment

# Core Features

1. Meetup Browsing
ผู้เล่นสามารถค้นหาตี้บอร์ดเกมที่เปิดรับคนอยู่ โดยกรองตามร้าน วันเวลา จำนวนคนที่ขาด ระดับผู้เล่น หรือประเภทเกม เช่น party, strategy, co-op, beginner-friendly

2. Create Meetup
ผู้เล่นสามารถสร้างตี้ใหม่ได้ โดยเลือกคาเฟ่ วันที่ เวลา จำนวนผู้เล่นขั้นต่ำและสูงสุด และเลือกรูปแบบตี้ได้ 2 แบบ:

    - Specific Game Meetup ตี้ที่ระบุเกมชัดเจน เช่น Catan, Werewolf, Splendor
    - Open Boardgame Meetup ตี้ที่ยังไม่ระบุเกม เหมาะสำหรับคนที่อยากเล่นบอร์ดเกมแต่ยังไม่รู้จะเล่นอะไร
   สามารถให้พนักงานร้านช่วยแนะนำเกมได้

3. Join Meetup
ผู้เล่นสามารถเข้าร่วมตี้ที่ยังมีที่ว่างได้ เมื่อกดเข้าร่วม ระบบจะสร้าง booking และให้ผู้เล่นจ่ายเงินมัดจำเพื่อยืนยันสิทธิ์เข้าร่วม

4. Deposit Payment
ระบบจำลองการจ่ายเงินมัดจำผ่าน payment gateway เช่น mock payment, Omise หรือ Stripe โดยในโปรเจค MVP สามารถทำเป็น mock payment ก่อน เพื่อให้เสร็จภายใน 8 สัปดาห์

5. QR Code Confirmation
เมื่อผู้เล่นจ่ายมัดจำสำเร็จ ระบบจะออก QR Code สำหรับใช้ยืนยันตัวตนเมื่อไปถึงร้าน พนักงานร้านสามารถสแกน QR Code เพื่อเปลี่ยนสถานะ booking เป็น checked_in

6. Cafe Dashboard
เจ้าของร้านสามารถดูรายการตี้ที่เกิดขึ้นในร้านของตนเอง ดูโต๊ะที่ถูกจอง จำนวนผู้เล่น และสถานะการจอง เช่น confirmed, checked_in, flaked

7. Board Game Catalog
ร้านสามารถแสดงรายการบอร์ดเกมที่มีในร้าน เพื่อให้ผู้เล่นเลือกตี้จากเกมที่สนใจได้ แต่ระบบไม่ควรบังคับให้ทุกตี้ต้องเลือกเกม เพราะผู้เล่นใหม่บางคนอาจไม่รู้จักเกมมาก่อน

8. Booking Status Tracking
ระบบมีสถานะการจอง เช่น:
    - pending_payment
    - confirmed
    - checked_in
    - completed
    - flaked
    - cancelled
สถานะเหล่านี้ช่วยให้ทั้งผู้เล่นและร้านเห็นความคืบหน้าของการจองอย่างชัดเจน
