# BoardGame Meetup Cafe App

## 1. Describe the human problems
- ความยากในการหาเพื่อนเล่น: ผู้เล่นที่อยากเล่นเกมเฉพาะทางหรือเกมที่ต้องใช้ผู้เล่นจำนวนมากมักหา "ตี้" หรือกลุ่มที่ว่างตรงกันได้ยาก
- การจัดการนัดหมายที่ไร้ระเบียบ: การนัดผ่านแชทกลุ่มทั่วไปมักทำให้ข้อมูลกระจัดกระจาย ไม่ทราบจำนวนผู้เล่นที่แน่นอน หรือเกิดปัญหาการนัดแล้ว "เท" (No-show)
- ขาดพื้นที่เชื่อมต่อระหว่างร้านกับลูกค้า: ร้านคาเฟ่บอร์ดเกมมักมีปัญหาในการสื่อสารกิจกรรมใหม่ๆ หรือเกมใหม่ๆ ให้ถึงตัวลูกค้าโดยตรง ทำให้ยากต่อการบริหารจัดการจำนวนโต๊ะว่างและรอบการเล่น

## 2. Describe how your solution solves these problems
- สร้างระบบ Community & Matchmaking: ผู้ใช้สามารถสร้างหรือค้นหา "นัดเล่น" (Meetups) ตามความสะดวกของเวลาและเกมที่สนใจ ทำให้การรวมกลุ่มมีประสิทธิภาพและตรงเป้าหมาย
- ระบบจัดการการจองที่แม่นยำ: แทนการใช้แชท แอปจะทำหน้าที่เป็นแพลตฟอร์มหลักที่มีสถานะการยืนยันเข้าร่วม (Confirmed/Pending) ช่วยลดปัญหาการเทนัดและช่วยให้เจ้าของร้านคาเฟ่ทราบจำนวนคนจองที่แน่นอน
- ช่องทางโปรโมทคาเฟ่โดยตรง: ร้านคาเฟ่สามารถลงประกาศกิจกรรมพิเศษ หรือแจ้งรายการเกมใหม่ๆ ในแอป เพื่อกระตุ้นให้ลูกค้าเข้ามาใช้บริการได้โดยตรง
- ระบบจัดการโปรไฟล์และทักษะ: ผู้เล่นสามารถระบุระดับความเชี่ยวชาญในเกม ทำให้ผู้เล่นใหม่และผู้เล่นเก่าสามารถจัดกลุ่มเล่นร่วมกันได้อย่างเหมาะสม ลดความกดดันของผู้เล่นใหม่

## 3. Describe your E-commerce business idea including core features

- Meetup Booking System (Core Feature): ระบบจองโต๊ะและจัดนัดเล่นแบบระบุสถานที่ (คาเฟ่) วันที่ และจำนวนคน โดยใช้โครงสร้างข้อมูลแบบ Hybrid (Referencing ร้าน/ผู้จัด + Embedding ข้อมูลผู้ร่วมงาน)
- Cafe Dashboard: ระบบหลังบ้านสำหรับคาเฟ่เพื่อบริหารจัดการคิวการจองโต๊ะ และดูภาพรวมของนัดหมายที่จะเกิดขึ้นในร้านตนเอง
- Game Inventory Catalog: รายการบอร์ดเกมของแต่ละคาเฟ่ที่เชื่อมโยงกับระบบนัดหมาย ทำให้ผู้เล่นเลือกคาเฟ่จาก "เกมที่มีให้บริการ" ได้
- Rating & Review System: ระบบรีวิวทั้งในส่วนของคาเฟ่ (บรรยากาศ/อาหาร) และผู้จัด (การจัดการ/ความสนุก) เพื่อสร้างความเชื่อมั่นให้กับคอมมูนิตี้
- Monetization Feature (ตัวเลือกทางธุรกิจ): ระบบขายตั๋วเข้าร่วมกิจกรรม (Ticket Sales) หรือการเก็บค่าธรรมเนียมการจองโต๊ะผ่านแอปพลิเคชัน เพื่อเป็นรายได้หลักของแพลตฟอร์ม

## 1. Describe the human problems
ปัญหาในฝั่งของผู้เล่น (Players) และเจ้าของร้าน (Cafe Owners) มีความสอดคล้องกันดังนี้
- The "Flake" Culture (ปัญหานัดแล้วล่ม): คนเล่นบอร์ดเกมมักเจอปัญหาชวนเพื่อนแล้วโดนเทในนาทีสุดท้าย ทำให้ตี้ล่ม หรือจำนวนคนไม่ครบจนไม่สามารถเล่นเกมที่อยากเล่นได้ (บอร์ดเกมหลายเกมต้องการจำนวนคนทีเฉพาะเจาะจง)

- The Stranger Danger & Social Anxiety: การจะเดินไปขอแจมตี้กับคนแปลกหน้าหน้างานที่ร้าน (Walk-in) แฝงไปด้วยความอึดอัด (Awkward) ไม่รู้ว่าตี้ล่วงหน้าเต็มไหม หรือพวกเขาเล่นเกมแนวที่เราชอบรึเปล่า

- Cafe Table Waste (ปัญหาของร้าน): ร้านบอร์ดเกมเสียโอกาสทางรายได้เมื่อมีคนโทรมาจองโต๊ะใหญ่ไว้ แล้วล่มหรือไม่มาตามนัด ทำให้ร้านเสียโต๊ะนั้นไปฟรีๆ โดยเฉพาะในวันเสาร์-อาทิตย์ที่ลูกค้าแน่น

## 2. Describe How Your Solution Solves These Problems
แอปพลิเคชันนี้จะทำหน้าที่เป็น "Platform Matchmaker & Booking System" ที่เปลี่ยนพฤติกรรมคนเล่นบอร์ดเกมให้มีวินัยและเข้าถึงง่ายขึ้น:
- Financial Commitment (แก้ปัญหาโดนเทด้วยเงินมัดจำ): ระบบบังคับให้ผู้ร่วมตี้ต้องจ่ายเงินมัดจำล่วงหน้า หากมาตามนัด เงินส่วนนี้จะแปลงเป็น Voucher/ส่วนลดสำหรับซื้อเครื่องดื่มหรือค่าบริการในร้านทันที แต่ถ้า "เบี้ยว" เงินจะถูกริบ (Non-refundable) ทำให้ทุกคนมีความรับผิดชอบในการมาสูงขึ้น

- Workspace-style Slot Booking: เปลี่ยนการหาตี้ให้เหมือนการจอง Co-working space ผู้ใช้สามารถเห็นได้ทันทีว่า วันนี้ที่ร้าน Board Game Everyday มีตี้ไหนเปิดอยู่บ้าง ต้องการคนกี่คน เล่นเกมแนวไหน และเหลือที่ว่างกี่ที่ สามารถกด Join และจ่ายมัดจำล่วงหน้าได้ทันที

- Win-Win Ecosystem: * ผู้เล่น: ได้เล่นเกมที่ชอบชัวร์ๆ สบายใจ ไม่โดนเท

    --> ร้านค้า: การันตีรายได้ขั้นต่ำจากค่ามัดจำ (แม้ลูกค้าจะเบี้ยว ร้านก็ยังได้เงินริบ หรือถ้าลูกค้ามา ร้านก็ได้ขายเครื่องดื่ม/อาหารเพิ่ม)

## 3. Describe Your E-Commerce Business Idea Including Core Features
- เป็นแพลตฟอร์มตัวกลาง (Marketplace) ที่เก็บค่าธรรมเนียมระบบ (Transaction Fee) เล็กน้อยจากทุกการจอง หรือเก็บค่าบริการระบบจอง/ระบบจัดการหลังบ้าน (SaaS) จากทางร้านบอร์ดเกม

# Scope

1. ฝั่งผู้เล่นทั่วไป (User / Player)
- Browse & Filter คาเฟ่และตี้: ค้นหาร้านบอร์ดเกมใกล้เคียง หรือเลือกฟิลเตอร์ตาม "เกมที่อยากเล่น", "ช่วงเวลา", และ "จำนวนคนที่ขาด"
- Create/Join Table (สร้างหรือเข้าร่วมตี้): * ผู้เล่น A สร้างตี้: ระบุร้าน, เกมที่จะเล่น, เวลา, และจำนวนคนที่ต้องการ
    --> ผู้เล่น B มาขอจองสล็อตที่ว่าง
- Payment & Deposit System (หัวใจหลัก): เชื่อมต่อ Payment Gateway (จำลองขึ้นมา เช่น Omise หรือ Stripe API) เพื่อตัดเงินมัดจำ และออก QR Code Confirmation สำหรับไปสแกนหน้าร้าน
- My Bookings & Wallet: หน้าประวัติการจอง และส่วนหน้ารวมคูปองส่วนลดเครื่องดื่มที่ได้จากการไปเล่นจริง

2. ฝั่งเจ้าของร้าน (Cafe Owner / Admin)
- Shop Profile Management: หน้าสำหรับใส่ข้อมูลร้าน, โลเคชัน, เวลาเปิด-ปิด และรายการบอร์ดเกมที่มีในร้าน (Board Game Library)
- Table & Slot Management: ระบบดูว่าวันไหน มีตี้ของลูกค้าจองเข้ามาบอร์ดไหนบ้าง เพื่อจัดสรรโต๊ะในร้านได้ถูกต้อง
- QR Code Scanner: หน้าสำหรับให้ร้านค้าใช้สแกน QR Code ของลูกค้าเมื่อมาถึงร้าน เพื่อเปลี่ยนสถานะตี้เป็น "Completed" และปลดล็อคคูปองส่วนลดเครื่องดื่มให้ลูกค้า

3. Database Schema Ideas (Back-end)
- Users (เก็บข้อมูลคนเล่น)
- Cafes (เก็บข้อมูลร้านค้าบอร์ดเกม)
- BoardGames (รายชื่อเกม)
- Parties/Tables (เก็บข้อมูลตี้ที่ตั้งขึ้น: สถานะตี้, เวลา, จำนวน Max)
- Bookings (เก็บ Data การจ่ายเงินมัดจำ, Status: Pending / Attended / Flaked)
